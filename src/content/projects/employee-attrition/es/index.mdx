# IBM HR Analytics — Análisis de Rotación de Empleados

## Introducción

La rotación de empleados es uno de los retos más comunes que enfrentan los departamentos de Recursos Humanos. Perder empleados con experiencia afecta la productividad, incrementa los costos de reclutamiento y a menudo revela problemas organizacionales más profundos que no son visibles de inmediato.

Este proyecto explora el dataset de IBM HR Analytics Employee Attrition con el objetivo de identificar patrones asociados a la rotación de personal a través de un flujo de trabajo estructurado de análisis de datos.

En lugar de enfocarse únicamente en producir visualizaciones, el proyecto está diseñado para demostrar el proceso analítico completo — desde entender el problema de negocio hasta comunicar hallazgos accionables. Cada etapa se documenta como parte de este caso de estudio, permitiendo al lector seguir tanto la implementación técnica como el razonamiento detrás de cada decisión.

El proyecto también sirve como un ejercicio práctico para combinar prácticas de ingeniería de software con análisis de datos. En lugar de depender de un solo notebook que contenga cada paso, la funcionalidad reutilizable se separa en módulos de Python, mientras los notebooks se mantienen enfocados en las preguntas de negocio y el razonamiento analítico.

---

# Contexto de Negocio

El departamento de Recursos Humanos de una empresa ha observado un número creciente de empleados que abandonan voluntariamente la organización. Aunque hay información demográfica, características del puesto y métricas de satisfacción laboral disponibles, no existe un entendimiento claro de qué factores están más fuertemente asociados con la rotación.

Sin este conocimiento, las decisiones sobre retención de empleados se vuelven reactivas en lugar de proactivas.

El propósito de este análisis no es construir un modelo predictivo de machine learning, sino entender los datos existentes, identificar relaciones significativas entre variables, y comunicar hallazgos que puedan apoyar futuras decisiones de negocio.

Este caso de estudio aborda el problema desde la perspectiva de un Data Analyst trabajando dentro de una organización, donde entender el problema de negocio es tan importante como escribir código.

---

## Verificando Valores Faltantes

Los valores faltantes frecuentemente representan una de las mayores fuentes de error analítico.

Por esa razón, se verifican explícitamente antes de realizar cualquier transformación.

```python
missing_values = (
    df.isnull()
      .sum()
      .sort_values(ascending=False)
)

missing_values = missing_values[missing_values > 0]
```

El dataset no contiene valores faltantes.

Debido a que no se requiere imputación, las etapas de preprocesamiento posteriores pueden enfocarse en simplificar el dataset en lugar de reparar información incompleta.

---

## Buscando Registros Duplicados

Los registros duplicados de empleados podrían sesgar los análisis subsecuentes.

Esta organización mejora la legibilidad, a la vez que hace que el flujo analítico sea más fácil de mantener conforme el proyecto crece.

El proyecto también ha reforzado el valor de abordar el análisis de datos desde una perspectiva de negocio.

En lugar de comenzar con visualizaciones o técnicas estadísticas, cada etapa inicia preguntando qué problema de negocio necesita resolverse.

Este enfoque guía naturalmente tanto las consultas SQL como el análisis exploratorio, resultando en hallazgos que son más fáciles de comunicar y más relevantes para quienes toman decisiones.

# Mejoras Futuras

Aunque el objetivo actual es completar el flujo analítico usando SQLite, el proyecto se ha diseñado para soportar futuras iteraciones.

Las posibles mejoras incluyen:

- Migrar de SQLite a PostgreSQL.
- Expandir el análisis en SQL con consultas más avanzadas.
- Introducir pruebas de hipótesis estadísticas.
- Construir dashboards interactivos usando Power BI.
- Comparar los enfoques de SQL y pandas para análisis seleccionados.
- Explorar modelos predictivos después de completar el análisis descriptivo.
- Publicar notebooks reutilizables con fines educativos.

El enfoque actual, sin embargo, sigue siendo completar un análisis descriptivo robusto antes de introducir técnicas analíticas más avanzadas.

# Conclusión

Este proyecto busca demostrar considerablemente más que dominio de pandas.

Documenta el proceso analítico completo que sigue un Data Analyst, comenzando con un problema de negocio y terminando con hallazgos accionables.

Al combinar una organización estructurada del proyecto, módulos de Python reutilizables, bases de datos relacionales, análisis en SQL y visualización exploratoria, el caso de estudio refleja un flujo de trabajo que se asemeja mucho más al análisis de datos profesional que a un notebook tradicional aislado.

Conforme se completen etapas adicionales, este documento evolucionará junto con el proyecto, registrando no solo la implementación técnica, sino también las decisiones, los trade-offs y los aprendizajes a lo largo de su desarrollo.
